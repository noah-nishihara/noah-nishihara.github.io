Once logged in, click the "+" icon in top right ‚Üí "New repository"

3

Add Your HTML File

In your new repository:

  1. Click "Add file" ‚Üí "Create new file"
  2. Name it: index.html
  3. Copy and paste the HTML template I created earlier
  4. Scroll down and click "Commit new file"

‚ö†Ô∏è Important:

The file MUST be named index.html - this is the default homepage GitHub Pages looks for.

4

Enable GitHub Pages

  1. Click on "Settings" tab in your repository

Method 2: GitHub Desktop (Recommended for Multiple Pages)

1

Download GitHub Desktop

Download from: desktop.github.com

  • Available for Windows and Mac
  • Install and sign in with your GitHub account
2

Create Repository

In GitHub Desktop:

  1. File ‚Üí New Repository
  2. Name: your-username.github.io
  3. Choose a local path (e.g., Documents/GitHub/)
  4. Check "Initialize this repository with a README"
  5. Click "Create Repository"
3

Add Your Files

  1. Click "Show in Explorer" (Windows) or "Show in Finder" (Mac)
  2. Create a file named index.html
  3. Paste in your HTML code and save
  4. Return to GitHub Desktop

üí° Pro Tip:

You can now drag and drop files directly into this folder! Add CSS files, images, additional HTML pages - they'll all appear in GitHub Desktop ready to publish.

4

Commit and Push

  1. In GitHub Desktop, you'll see your changes listed
  2. Add a commit message: "Initial site launch"
  3. Click "Commit to main"
  4. Click "Publish repository" (top right)
  5. Make sure "Keep this code private" is UNCHECKED
  6. Click "Publish repository"
5

Enable GitHub Pages

  1. Click "View on GitHub" in GitHub Desktop
  2. In your browser, go to Settings ‚Üí Pages
  3. Source: "Deploy from a branch"
  4. Branch: "main", Folder: "/ (root)"
  5. Click "Save"

üéâ Done!

Your site is live at: https://your-username.github.io

Future updates: Just edit files locally, commit in GitHub Desktop, and click "Push origin" - changes go live in seconds!

üîß Common Issues

Site shows 404 error

Solution: Wait 1-2 minutes after enabling Pages. It takes time to build. Check that your file is named exactly index.html (lowercase, no spaces).

Site shows README instead of my page

Solution: Make sure you have an index.html file. GitHub Pages prioritizes index.html over README.md

Changes not appearing

Solution: Clear your browser cache (Ctrl+Shift+R on Windows, Cmd+Shift+R on Mac). Sometimes changes take 30-60 seconds to propagate.
